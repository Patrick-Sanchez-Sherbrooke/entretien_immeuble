Note: dans la base de données 'supabase', on a ajouté le champ 'created_by' et 'execution_note' dans la table 'tasks'

A- Écran de connexion: on ne voit pas le message d'erreur dans le carré rouge

B- Ecran de gestion des immeubles et écran de gestion des utilisateurs: faire apparaitre le texte du carré dans le titre d'une différente couleur

C- Ecran de liste des tâches: sous l'étage, mettre:
1- icône + "Description:' + description de la tâche. Si la description est trop longue ppour la mettre sur la ligne, la tronquer en mettant des '...' à la fin
2- icône + "Créée le:" + date de création si elle existe (created_by dans la table) + nom du créateur (justifié à droite) en enlevant le 'point' devant le nom
3- icône + "Planifiée le:" + date de planification si elle existe
4- icône + "Terminée le:" + date de terminaison si elle existe + nom de l'exécutant (justifié à droite)

D- Ecran de détail d'une tâche et de modification d'une tâche: mettre une execution_note (si elle existe) sous le nom de l'exécutant

E- Modification de tâche: quand on passe du statut de 'terminée' à 'en cours', on doit supprimer la photo + date d'exécution

F- Dans la table tasks, stocker l'id l'immeuble au lieu du nom et modifier:
1- liste des tâches: tenir compte de ce stockage pour trouver le nom de l'immeuble 
2- détail 'une tâche: tenir compte de ce stockage pour trouver le nom de l'immeuble
3- Modification d'une tâche: tenir compte de ce stockage pour trouver le nom de l'immeuble
4- Rapports: tenir compte de ce stockage pour trouver le nom de l'immeuble

G- Ajouter un rôle de 'Planificateur' dans la création ou la modification d'un utilisateur. Il a les mêmes privilèges que les utilisateurs mais ne peut pas exécuter une tâche (pas possible de mettre de date d'exécution ni de photo ni de note).
